# Pre-Sales Automation Patterns

## The Core Idea

Pre-sales automation should remove drag, not remove judgment.

The best systems do three things well:

- reduce repetitive work
- preserve context between steps
- make next actions more obvious

## Common Failure Pattern

Teams often automate fragments:

- a form here
- a template there
- a script for one narrow handoff

That creates activity, not a system.

## Better Pattern

Start with the operating model:

1. identify where time and context are leaking
2. map the systems and handoffs involved
3. define the minimum shared layer that improves continuity
4. only then decide where prompts, scripts, and automations belong

## Measurement

Good metrics include:

- manual time reduced
- handoff quality improved
- response-cycle speed increased
- transferability to other programs
